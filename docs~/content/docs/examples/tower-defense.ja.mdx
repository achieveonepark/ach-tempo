title: タワーディフェンス

---

> **なぜ合うか:** 2つが同時にハマります — ディレクターのウェーブ・ペーシング(Left 4 Dead)と、
> 凍結・感電が敵の群れへ広がる状態異常の連鎖(BotW の電気チェーンそのまま)。

## 部品の対応

| Tempo | タワーディフェンスでは |
|---|---|
| `TensionModel` | 防衛線の圧力（漏れた敵数、コア HP） |
| `DirectorBrain` の局面 | 偵察(BuildUp) → ラッシュ/特殊敵(Peak) → 立て直し(Relax) |
| `SeedAction` | `SeedColdSnap`（凍結場）· `ChargeStorm`（感電チェーン）· `SpawnRusher` |
| `WorldReaction` | 状態伝播：凍結/感電が罹患した敵から隣の敵へ広がる（要素反応のまま） |
| 安全装置の上限 | 同時の敵/弾の上限 = モバイルのフレーム防御 |

## 状態連鎖こそ要素反応

BotW の「電気が金属を伝って広がる」ルールが、ディフェンスの「感電が濡れた/金属の敵の群れへチェーン」に
そのまま移ります。`ReactionRule` 表を敵の状態異常として読むだけです。

```csharp
// 「感電が濡れた敵の群れへチェーン（隣の敵へ飛ぶ）」
new ReactionRule {
    Trigger = ElementTag.Electric,
    TargetMaterials = "WetEnemy;MetalEnemy",
    Effect = StateEffect.Discharge,   // チェーン感電イベントを発火
    Probability = 0.8f,
};
```

防衛線が楽すぎれば（緊張度が低い）ディレクターがラッシュ/特殊敵を撒いて締め、手に負えなければ（高い）
息継ぎを与えます。プレイヤーが敷いた凍結場へ感電を誘導する「要素コンボ」も、同じ表から自然に生まれます。

## モバイルのチェックポイント

- **コンボが手応え。** 凍結 → 感電のような連鎖をルール表に置けば、新コンボをデータでライブ追加。
- **ウェーブ・ペーシング＝難易度曲線。** ステージを手で組まなくてもディレクターが圧力を調整。
- **上限＝性能。** 同時の敵/弾の上限で、終盤の物量戦でもフレームを安定維持。
