## Tempo がすること

ゲームが自分で危機的状況を作り出すシステムです。**演出担当（ディレクター）** が火・雨・風といった
条件をそっと撒いておくと、**ワールドリアクション** 側がそれを受け取り、火が燃え広がるように状況を
育てます。一場面ずつ手で組まなくても、毎回ちがう状況がひとりでに起こります。

2つのゲームから着想を得ています。

- **Left 4 Dead** — 状況の強弱（テンポ）を自動で調整。
- **ゼルダの伝説 BotW** — 火・水・電気が決まったルールで互いに反応。

> **名前が Tempo の理由:** このパッケージが結局やるのは、ゲームの緊張「テンポ（強弱）」を調整することだからです。

## ひと回りの流れ

```text
[ディレクター]  ──条件を撒く(火/雨/風/雷)──▶  [ワールドリアクション]
   ▲                                            │
   │  緊張度を受け取る                          │ 結果を計算 (延焼/鎮火/通電)
   │  (プレイヤーの脅威 + 周囲で起きていること)  ▼
   └──────────────  [緊張度モデル]  ◀── プレイヤーの脅威 / いま起きている事象
```

毎フレーム、3つの部分は必ず **緊張度 → ディレクター → ワールド** の順で回ります。

| 部分 | クラス | 役割 |
|---|---|---|
| ワールドリアクション | `WorldReaction` | 格子上でルール通り延焼し、いま起きている事象の一覧を更新 |
| ディレクター | `DirectorBrain` | 緊張度を見て局面を決め、予算の範囲で条件を撒く |
| 緊張度モデル | `TensionModel` | プレイヤーの脅威＋周囲の事象を 0〜1 の数値ひとつに合成 |

## 基本原則

1. **役割を2つに分ける。** ワールドリアクションはディレクターの存在を一切知らない。ルール通り広がるだけ。
2. **ディレクターは敵を直接出さない。** 代わりに条件だけを撒く（上流に火・雨・風・雷雲）。
3. **ディレクターが見るのは数値ひとつ** — 緊張度（0〜1）。プレイヤーの脅威と周囲の事象を合わせた値。
4. **数値はすべて外で調整。** ルール表・しきい値・重みはデータに切り出し、コードなしで触る。
