> **为何契合：** 两点同时命中 —— 导演的波次节奏（求生之路），以及冻结/感电在敌群中蔓延的
> 状态异常连锁（旷野之息的电链，原样照搬）。

## 部件对应

| Tempo | 在塔防中 |
|---|---|
| `TensionModel` | 防线压力（漏过的敌人数、核心血量） |
| `DirectorBrain` 阶段 | 侦察(BuildUp) → 猛攻/特殊怪(Peak) → 整顿(Relax) |
| `SeedAction` | `SeedColdSnap`（冻结场）· `ChargeStorm`（感电连锁）· `SpawnRusher` |
| `WorldReaction` | 状态传播：冻结/感电从中招的敌人跳向邻近敌人（沿用要素反应） |
| 安全上限 | 同时敌人/弹幕上限 = 移动端帧率防护 |

## 状态连锁就是要素反应

旷野之息「电沿金属传导蔓延」的规则，直接映射到防御游戏「感电在潮湿/金属敌群中连锁」。
只需把 `ReactionRule` 表当作敌人的状态异常来读。

```csharp
// “感电在潮湿敌群中连锁（跳向邻近敌人）”
new ReactionRule {
    Trigger = ElementTag.Electric,
    TargetMaterials = "WetEnemy;MetalEnemy",
    Effect = StateEffect.Discharge,   // 触发连锁感电事件
    Probability = 0.8f,
};
```

防线太轻松时（紧张度低）导演铺设猛攻/特殊怪来收紧；招架不住时（高）给出喘息。
把感电引到玩家铺下的冻结场上的「要素连招」，也会从同一张表里自然产生。

## 移动端要点

- **连招即手感。** 把冻结 → 感电这类连锁放进规则表，就能以数据在线上新增连招。
- **波次节奏＝难度曲线。** 无需手工编排每一关，导演就能调节压力。
- **上限＝性能。** 用同时敌人/弹幕上限，在后期海量战斗中也保持帧率稳定。
