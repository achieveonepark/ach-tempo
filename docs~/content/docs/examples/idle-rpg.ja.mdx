> **なぜ合うか:** 放置系は「プレイヤーが見ていない間」にも起伏がないと、また開いてもらえません。
> ディレクターが緊張曲線を作り、「戻ってくる価値のある出来事」をペーシングします。

## 部品の対応

| Tempo | 放置系では |
|---|---|
| `TensionModel` | セッション関与/脅威の曲線（平坦＝離脱、起伏＝復帰） |
| `DirectorBrain` の局面 | 平穏(BuildUp) → ボス出現/襲撃ラッシュ(Peak) → 報酬精算・休息(Relax) |
| `SeedAction` | `SpawnRaidBoss` · `TriggerGoldRush` · `AmbushEvent` |
| `WorldReaction` | 資源・生態の連鎖（鉱脈枯渇→価格変動、モンスター繁殖→狩場過密）をルール表で |
| 前兆 | 「まもなくボス出現」通知 = **プッシュ通知での復帰誘導** |

## オフライン進行を一括精算

放置系は毎フレーム回す代わりに、復帰時に「空けていた時間」を一括精算することが多いです。
3つの `Tick` をゲーム時間（累積経過）で大きな歩幅（例: 1分 = 1ティック）で回し、留守の間の局面変化と
出来事を圧縮再生します。ディレクターは「いつボスが出たか」などの起伏だけを作り、実際の戦闘計算は
放置シミュレーションが受けます。

```csharp
// 復帰時: 空けていた時間を大きな歩幅で圧縮再生
void CatchUp(float secondsAway)
{
    const float step = 60f; // 1分 = 1ティック
    for (float t = 0; t < secondsAway; t += step)
    {
        _tension.Tick(playerPos, _world, step);
        _director.Tick(_world, _tension, playerPos, step); // ボス/襲撃の出来事をペーシング
        _world.Tick(step);
    }
    // 蓄積した ActiveEvents を「留守中こんなことが」レポートで見せる。
}
```

## モバイルのチェックポイント

- **起伏＝リテンション。** 平坦な自動進行は離脱を招く。ディレクターがボス/ラッシュで復帰動機を作る。
- **前兆＝プッシュ。** 「30分後にレイドボス」予告をプッシュ通知に繋いで復帰を誘導。
- **電池に優しい。** 毎フレームではなく復帰時の圧縮再生 → バックグラウンド負荷ゼロ。
