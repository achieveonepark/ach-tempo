## Tempo 做什么

这是一套让游戏自己制造危机局面的系统。**导演（演出）** 只是悄悄铺下火、雨、风之类的条件，
**世界反应** 一侧便接过这些条件，像火势蔓延一样把局面放大。不必逐个场景手工编排，
每一次都会自动发生不同的状况。

灵感来自两款游戏：

- **求生之路（Left 4 Dead）** —— 自动调节局势的强弱（节奏）。
- **塞尔达传说：旷野之息** —— 火、水、电按固定规则彼此反应。

> **为何叫 Tempo：** 这个包最终做的事，就是调节游戏紧张感的「节奏（强弱）」。

## 一轮循环

```text
[导演]  ──铺设条件(火/雨/风/雷)──▶  [世界反应]
   ▲                                  │
   │  取回紧张度                      │ 计算结果 (蔓延/熄灭/通电)
   │  (玩家威胁 + 周围正在发生的事)    ▼
   └──────────────  [紧张度模型]  ◀── 玩家威胁 / 当前事件
```

每一帧，三个部分始终按 **紧张度 → 导演 → 世界** 的顺序运行。

| 部分 | 类 | 职责 |
|---|---|---|
| 世界反应 | `WorldReaction` | 在网格上按规则蔓延，更新当前事件列表 |
| 导演 | `DirectorBrain` | 看紧张度决定阶段，在预算内铺设条件 |
| 紧张度模型 | `TensionModel` | 把玩家威胁＋周围事件合成为一个 0~1 的数 |

## 核心原则

1. **把两个角色分开。** 世界反应完全不知道导演的存在，只按规则蔓延。
2. **导演从不直接生成敌人。** 只铺设条件（在上游放火、雨、风、雷云）。
3. **导演只看一个数** —— 紧张度（0~1）：玩家威胁与周围事件的合成值。
4. **所有数值都在代码之外调。** 规则表、阈值、权重都抽成数据，策划无需写代码即可调整。
