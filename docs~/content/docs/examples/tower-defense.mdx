> **왜 잘 맞나:** 두 가지가 동시에 들어맞습니다 — 디렉터의 웨이브 페이싱(Left 4 Dead)과,
> 빙결·감전이 적 무리로 번지는 상태이상 연쇄(야숨 전기 체인 그대로).

## 부품 매핑

| Tempo | 타워 디펜스에서 |
|---|---|
| `TensionModel` | 방어선 압박 (샌 적 수, 코어 체력) |
| `DirectorBrain` 국면 | 정찰(BuildUp) → 러시/특수몹(Peak) → 정비(Relax) |
| `SeedAction` | `SeedColdSnap`(빙결장) · `ChargeStorm`(감전 체인) · `SpawnRusher` |
| `WorldReaction` | 상태이상 전파: 빙결·감전이 붙은 적에서 옆 적으로 번짐(요소 반응 그대로) |
| 안전장치 상한 | 동시 적/투사체 상한 = 모바일 프레임 방어 |

## 상태이상 연쇄가 곧 요소 반응

야숨에서 '전기가 금속을 타고 번지는' 규칙이, 디펜스에선 '감전이 젖은/금속 적 무리로 체인' 으로
그대로 옮겨집니다. `ReactionRule` 표만 적의 상태이상으로 해석하면 됩니다.

```csharp
// "감전이 젖은 적 무리로 체인 (옆 적에게 번짐)"
new ReactionRule {
    Trigger = ElementTag.Electric,
    TargetMaterials = "WetEnemy;MetalEnemy",
    Effect = StateEffect.Discharge,   // 체인 감전 이벤트 발생
    Probability = 0.8f,
};
```

디렉터는 방어선이 너무 수월하면(긴장도 낮음) 러시/특수몹을 깔아 조이고, 벅차면(높음) 숨틈을 줍니다.
플레이어가 깔아둔 빙결장 위로 감전을 유도하는 '요소 콤보' 도 같은 표에서 자연스럽게 나옵니다.

## 모바일 체크포인트

- **콤보가 손맛.** 빙결 → 감전 같은 연쇄를 규칙표로 두면 라이브에서 새 콤보를 데이터로 추가.
- **웨이브 페이싱으로 난이도 곡선.** 스테이지마다 손수 짜지 않아도 디렉터가 압박을 조절.
- **상한 = 성능.** 동시 적/투사체 상한으로 후반 물량전에서도 프레임 유지.
